### ⚙️ HTTPS 的加密过程（简化流程）
1. 客户端发起 HTTPS 请求
2. 服务端返回证书（含公钥）
3. 客户端验证证书是否可信
4. 客户端生成对称密钥，并用公钥加密发送给服务端
5. 服务端用私钥解密，获得对称密钥
6. 双方使用这个对称密钥进行加密通信

### 🏁 小结
| 类型    | 用途        | 加密对象       | 性能 |
| ----- | --------- | ---------- | -- |
| 非对称加密 | 密钥协商、身份验证 | 小数据（密钥）    | 慢  |
| 对称加密  | 数据传输加密    | 实际业务数据     | 快  |
| 数字证书  | 服务器身份验证   | 非加密，用于验证签名 | -  |
